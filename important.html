Dicas HTML/CSS

  • Arquivo html são geralmente nomeados com "index.html"
  • Texto em itálico: usamo o a tag <'em'>
  • Texto em negrito: usamo a tag <'strong'>
  • Tranformar stylo da fonte: usamos o comando 'font-weight:'
  • Leitura no CSS: é feita de cima para baixo, ou seja, se tivermos mais de um item igual a ultimá sobrescreve todas as anteriores.
    • Caso queira mais de um item igual basata adiciona-los na mesma linha com espç=aço separando.

      • Ex: tranform: scale(1.2) rotate(70deg)

      Estamos aplicando um aumento de 20% e uma rotação em 70° no elemento.

  • Imagem: caracteristicas da tag img.

    No HTML

    • uso <'img' src="caminho da imagem" alt="aparece caso a imagem não carregue corretamente ou não seja acessível para o usuário.">

    No CSS

  • Comportamento block: são tags que bloqueiam o conteúdo da linha inteira como li, lo.
    • Não permitem que alterem dimensões
    • Permite alterar espaçamento interno e externo
    • uso "display:block"
  • Comportamento inline São tags que não bloqueiam o conteúdo tod da linha, com img.
    • Permitem alterar dimensões
    • Não permitem alterar espaçamento interno e externo
    • uso "display:inline"
  • Comportamento inline-block
    • Ocupa somente o espaço do conteúdo.
    • Permitem alterar dimensões
    • Permitem alterar espaçamento interno e externo
    • uso "display:inline-block"
  • Alinhamento de elementos no CSS: por padrão todos os elementos no css são alinhados pela linha de baixo.
    • Podemos alterar o formato do alinhamento
    • Uso "vertical-align:"
  • Tag HEADER: foi uma tag criada no HTML5 que indica para o navegador que aquilo é o cabeçalho, ou seja a primeira informação que o usuário vai ver.
  • Importância das Classes: As classes criadas no Html tem o intuito de otimizar e reutilizar o stylo definido no css.
    • Assim, não iremos criar varios stylos desnecessário dentro do CSS.
    • Poderemos facilmente fazer a manutenção do código.
  • Tag ANCOR <'a'>: utilizamos para tornar realizável a navegação entre as páginas.
  • Tag <'nav'>: é uma tag criada no HTML5 para criar um estilo de navegação entre links do site
  • Espaço entre elementos: colocamos no item e não no texto ou link.
  • Navegador interferindo: É normal que o navegador mexa nas tags do HTM e nos stylos do CSS. Poŕem, na maioria das vezes atrapalha o controle do programador em relação ao site.

    Para resolvermos utilizamos um arquivo que resta todas essas interferências do navegador criado pela comunidade.

    • Utilizamos o arquivo reset.css
    • Sempre antes de importar o meu CSS.
  • Posição fixa : é a opção padrão do CSS.
    • É a posição natural quando cada elemento é criado.
    • Não pode mudar o ponto inicial
    • uso "position: static"
  • Posição Relativa: é uma das opções oferecidas pelo CSS.
    • O referêncial e a posição inicial/natural do objeto.
    • Podemos mudar a posição, de acordo com o referêncial.
    • Uso "position: relative"
  • Posição absoluta: é uma das opções oferecidas pelo CSS.
    • Conseguimos mudar o objeto de posição sem ele ele estar preso a uma posição inicial do proprio objeto.
    • O referencial será outro elemento.
    • Uso "position: absolute"
  • Espaçamento Interno x Espaço reservado: por padrão o navagador soma o espaço ocupado pelo objeto com seu espaçamento interno, "padding".

    Geralmento esse formato de calculo quebra a formatação desejada.

    • É muito comum trabalhar com unidades diferentes de dimensão e espaçamento.
    • Usamos "box-sizing: border-box", para garantir que o espaçamento interno seja calculado dentro do espaço reservado para o objeto.
  • Bordas: a borda tem 3 aspectos importantes

    Podemos declará-los separadamente ou juntos em uma só linha de código

    • Juntos

      Uso "border: 'tamanho' 'tipo' 'cor'"

    • Tamanho

      Podemos definir um tamanho só, ou definir o tamanho de lado da borda

      • "border-width" - tamanho em todos os lados
      • "border-top-width" - tamanho no topo
      • "border-left-width" - tamanho na esquerda
      • "border-right-width" - tamanho na direita
      • "border-bottom-width" - tamanho em baixo

      Podemos arredondar as bordas

      Antes do CSS3 era muito trabalhoso, usava-se uma imagem com as informações desejadas e adicionava borda a essa imagem

      Com o CSS3 é possível realizar cálculos para arredondar a borda, através de angulo

      • A medida que colocamos é o raio do circulo desenhado nos cantos
      • "border-radius" - em todos os cantos
      • "border-top-left-radius" - canto superior esquerdo
      • "border-top-right-radius" - canto superior direito
      • "border-bottom-left-radius" - canto inferior esquerdo
      • "border-bottom-right-radius" - canto inferior esquerdo

    • Tipo

      Temos varios stylos

      • "border-style: solid" - totalmente preenchido
      • "border-style: dashed" - tracejado
      • "border-style: dotted" - pontilhado

    • Cor

  • Comportamento de objetos com CSS: podemos utilizar o CSS para mapear o que o usuários está fazendo e definir um determinado comportamento para um determinado objeto.

    É definido também para todos os filhos, netos, etc... desse elemento

    Ações possíveis para o usuario

    • Hover: mouse em cima

      usamos "epecificação do obleto:hover{}" e depois definimos o que será feito.

      Mudar características do elemento

      Uso "especificação do elemento:hover nome característica"

    • Active: click com mouse precionado

      usamos "especificação do objeto:active{}" e depois definimos o que será feito.

  • Transição: utilizamos para suavizar mudanças estabelecidas por comportamentos.
    • Antes do CSS3 era feito só com o java script.
    • uso no CSS "transition: 'tempo de delay' e 'qual parte será realizada a transição'"
    • Se colocarmos "all" na parte que será realizada a transição, então aplicaremos a transição para todos.
  • Imagem como fundo: podemos definir uma imagem para ser o fundo de um elemento.
    • Se a imagem for pequena em relação a área do fundo, por padrão o CSS vai copiar e colar a imagem até ocupar todo o espaço.
    • uso "background: url('caminho da imagem');"

Transformções com CSS3

Podemos fazer algumas transformações apenas com CSS3.

  • uso "transform: "
  • uso "transform: scale('escala')" para aumentar ou diminuir um elemento usando escala.

    Mais fácil do que definir um before com todas as características do novo tamanho.

  • uso "transform: rotate('graus'deg)" para rotacionar um elemento usando graus.

Formulários

Usamos formulários quando precisamos coletar informações do usuário e enviá-las para um banco de dados, um email, etc.

  • Todos os campos de um formulário , input's label's, são INLINE, então não dá pra alterar espaçamentos interno e externos.
  • Tag's "p" e "div": não é profissional usar tag "p" e "div", pois prejudica a semântica do formulário.
    • "div" substitui por "fieldset"
    • "p" substitui por "legend"
  • Campos obrigatórios: utilizamos quando queremos que aquele campo seja obrigatoriamente preenchido pelo usuário.
    • uso na tag do "input" a palavra resevada required.
    • valido para todos os tipos de input's e textarea.
  • Sugestão de preenchimento / exemplo: usamos para mostra ao usuário o que e como deve ser preenchido aquele campo do formulário.
    • uso dentro da tag input 'placeholder = ""'
  • Opção padrão: podemos definir uma opção padrão pra ser marcada adicionando dentro da tag a palavra reservada "checked"
    • Funciona no input do tipo radio, checkbox.
  • Label e input: um para interligado que anda sempre junto.
    • Label: tag usada para indicar quais informações devem ser colocadas em cada input.
      • uso "for='id do input'" para estabelecer a conexão input<---->label
    • Input: tag usada para colocar caixinha de input.
      • Geralmente é único.
      • Sempre tem identificador, "id=".
      • Todo formulário tem um input do tipo "submit", não precisa de label.

        • Tipos input

        Site com dicas de como o mobile se comporta com cada tipo de input do HTML5
        1. type="text" - usado para caixa de texto
        2. type="radio" - usado para mostrar opções selecionaveis.
          • Por padrão podem ser selecionados todos os radios de uma vez.
          • Para obrigatoriamente selecionar apenas um precisamos adiconar "name=' '" em todos os radios do grupo.
        3. type="checkbox" - usado para opções selecionaveis, formato caixa.
        4. OUTROS
        5. type="submit" - usado para enviar as informações coletadas nos outros input.
          • Obrigatório em todo formulário.
          • "submit" será traduzido para a lingua que o usuário está utilizando.
          • Usamos type="submit" value="label fixa" se quisermos que o "submit" não seja traduzido.
    • Input e label: para criar ambos juntos basta colocar o 'input' dentro do 'label'
  • caixa de mensagem: Utilizada para designar uma espaço específico para texto maior.
    • uso a tag de conteúdo<'textarea'>
    • Pode ser acompanhada la tag 'label'.
    • cols="" - defini quanta colunas terá a caixa de texto, equivalente a quantas letras cabe em cada linha.
    • rows="" - defini quantas linhas terá a caixa de texto.
    • Para definir o tamanho da caixa de forma mais profissional usamos o CSS e não 'cols' e 'rols'.
  • Lista opções embutida: oferece varias opções pro usuário escolher, em formato de lista embutida.
    • uso a tag de conteúdo <'select'>
    • uso a tag de conteúdo <'option'> dentro do "select" para inserir cada opção.

hierarquia no CSS

Os marcadores dentro do CSS seguem essa ordem de "força/prioridade".

  1. CSS in line
  2. id - stronger
  3. class - intermediate
  4. tag - weaker

    5. Ou seja:

    • Nenhum style sobrepõe o style do css in line.
    • Caso esteja utilizando CSS externo, nada sobrepõe ao style do id.
    • É recomendável utilizar classes nos elementos pois, não é tão genérico a ponto de sempre ter que mudar, quando muda a tag, e nem é tão específico a ponto de funcionar somente em um elemento.

Tabelas

É um conjunto de linha e colunas.

    No HTML

  • Linhas: uso a tag de conteúdo "<'tr'>".
  • Colunas/células: uso a tag de conteúdo "<'td'>".
  • Cabeçalho: uso a tag de conteúdo <'thead'>.
  • Título do cabeçalho: uso a tag de conteúdo <'th'>
  • Corpo da tabela: uso a tag de conteúdo <'tbody'>
  • Para um código mais profissional invista na semântica da tabela, para torná-la mais intuitiva e estilizada.

    • Exemplo:

    Dia Horário
    Segunda-feira 8h às 18h
    Quarta-feira 8h às 18h
    Sexta-feira 8h às 18h
